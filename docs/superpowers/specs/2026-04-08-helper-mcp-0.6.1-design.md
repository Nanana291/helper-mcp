# helper-mcp 0.6.1 Design

## Goal

Ship a large follow-up release that expands helper-mcp in three coordinated directions: deeper Luau intelligence, a more connected brain platform, and stronger release automation.

The release keeps the 0.6.0 stabilization work intact, but adds higher-level commands and richer relationships so the server can explain code, memory, and workspace drift with more context and less noise.

## Scope

### In scope

- Add new Luau analysis commands that move beyond pattern counting into graph-aware and confidence-aware analysis.
- Add new brain commands that expose relationships, archiving, diffable recovery, and duplicate cleanup.
- Add new workspace commands that make diffs, restores, validation, audit, and release note generation first-class.
- Keep the current canonical tool names and underscore aliases stable for the 0.6.0 surface.
- Preserve fail-closed behavior for malformed pattern overrides.
- Keep release artifacts and regression proof for all new commands.

### Out of scope

- Breaking canonical names or aliases that exist in 0.6.0.
- Turning helper-mcp into a generic code runner or arbitrary transform engine.
- Deep control-flow or taint engines that cannot be covered by deterministic tests.
- UI changes unrelated to the new command surface.

## Architecture

### Shared Spine

The release shares one internal spine across all three pillars:

- `pattern/confidence engine`
  - Findings carry `severity` and `confidence`.
  - Heuristic results remain clearly marked as such.
- `snapshot/history layer`
  - Every write-path command leaves a recoverable artifact.
  - Temporal behavior is recorded in append-only history where possible.
- `regression gate`
  - Benign fixtures stay clean.
  - Changes in tool surface, counts, or aliases are explicitly tested.

This spine keeps the release large in scope, but bounded in behavior.

### Pillar 1: Luau Intelligence

Luau commands should expose richer structural knowledge, not just regex hits.

Planned commands:

- `luau.taint`
  - Track risky values from source to sink across a single file or workspace slice.
- `luau.flow`
  - Summarize data flow between locals, functions, and remote calls.
- `luau.handlers`
  - Map remote events/functions to likely handlers and missing coverage.
- `luau.surface`
  - Summarize the external surface of a script: remotes, callbacks, UI entry points, and high-risk sinks.
- `luau.refactor`
  - Return conservative refactor suggestions for high-confidence risks.
- `luau.modulegraph`
  - Build a module dependency graph and identify disconnected or overly shared modules.
- `luau.risk_score`
  - Produce a normalized score from risks, remote usage, cleanup gaps, and hot-loop pressure.
- `luau.diff_context`
  - Diff two Luau files with surrounding context and a concise explanation of what changed.

Design intent:

- Prefer analysis and suggestion over automatic rewriting.
- Keep confidence high enough that the new commands reduce noise instead of increasing it.
- Reuse the existing Luau parsing helpers and patterns infrastructure where possible.

### Pillar 2: Brain Platform

Brain commands should expose structure and recovery, not just raw notes.

Planned commands:

- `brain.graph`
  - Show note-to-note relationships from tags, source paths, and similarity.
- `brain.query_advanced`
  - Rank notes with richer filters for status, scope, tags, and time windows.
- `brain.link`
  - Explicitly connect notes so related lessons can be recovered together.
- `brain.archive`
  - Move old or superseded notes into an archive stream without deleting history.
- `brain.restore_diff`
  - Compare a previous brain snapshot to the current one and show what changed.
- `brain.prune_duplicates`
  - Find and consolidate likely duplicate notes using conservative similarity rules.

Design intent:

- Keep merges explicit and reversible.
- Preserve note history even when notes are archived or consolidated.
- Make the brain useful as a memory graph, not only a note list.

### Pillar 3: Release Automation

Workspace commands should make change review and rollback cheaper.

Planned commands:

- `workspace.diff`
  - Compare current workspace state to a baseline and summarize drift.
- `workspace.rollback`
  - Restore a workspace or file from a known-good snapshot.
- `workspace.validate`
  - Validate that the current workspace still satisfies the expected release shape.
- `workspace.release_notes`
  - Generate human-readable release notes from diffs, brain links, and snapshots.
- `workspace.audit`
  - Produce a consolidated audit for Luau risk, brain coverage, and release drift.
- `workspace.restore_snapshot`
  - Restore a named snapshot and verify the post-restore state.

Design intent:

- Use existing baseline and changelog mechanics instead of inventing a parallel system.
- Keep restore operations deterministic and file-scoped where possible.
- Generate release notes from actual state transitions, not prose-only summaries.

## File Boundaries

- `src/luau.mjs`
  - New Luau analysis helpers and formatter output.
  - Graph and score functions for Luau intelligence.
- `src/brain.mjs`
  - Relationship graph, archive, advanced query, diff, and duplicate cleanup.
- `src/workspace.mjs`
  - Diff, rollback, restore, validation, and release note generation.
- `src/core.mjs`
  - MCP tool registry, aliases, handlers, versioning, and resource exposure.
- `src/patterns.mjs`
  - Pattern override loading and category normalization.
- `src/metrics.mjs`
  - Trend snapshots if the new Luau/workspace commands emit metrics.
- `test/*.mjs`
  - Targeted behavior tests for each pillar plus compatibility coverage.
- `docs/releases/0.6.1/*`
  - Task lock, handoff, regression matrix, and delivery gate.

## Data Flow

### Luau flow

1. Read source text and workspace context.
2. Load project patterns with fail-closed overrides.
3. Analyze for remotes, handlers, dependencies, flow, and risk.
4. Attach confidence and severity.
5. Return structured output that can be consumed by repair and audit flows.

### Brain flow

1. Load notes and history.
2. Compute similarity, links, and archive state.
3. Expose recoverable diffs against a prior snapshot.
4. Consolidate duplicates only when the caller explicitly requests it.

### Workspace flow

1. Compare the current state against a baseline or snapshot.
2. Generate diff, audit, and release note artifacts.
3. Restore or rollback only through deterministic file and snapshot paths.
4. Confirm post-restore state before reporting success.

## Error Handling

- Malformed pattern overrides fall back to defaults.
- Queries with no matching notes or files return empty structured results rather than partial failures.
- Archive, prune, and restore operations should refuse destructive work without explicit targets.
- Restore and rollback operations should surface missing snapshot or baseline paths clearly.
- New commands should fail closed instead of guessing about workspace intent.

## Testing

### Luau tests

- Benign snippets remain below the risk threshold.
- Dependency and flow graphs return useful structure on small fixtures.
- `luau.refactor` should only suggest safe rewrites.
- `luau.diff_context` should preserve context around additions and removals.

### Brain tests

- Graph and advanced query commands should respect status, tags, and time windows.
- `brain.link` should create durable relationships.
- `brain.archive` should preserve history and counts.
- `brain.prune_duplicates` should require explicit application and preserve the surviving note.

### Workspace tests

- Diffs should match baseline drift in a temp workspace.
- Rollback and restore snapshot commands should round-trip a file or workspace.
- Release notes should reflect actual changed files and links.
- Audit output should include the current Luau and brain state.

### Regression proof

- Preserve `0.6.0` canonical behavior and aliases.
- Confirm the new commands are added without removing the existing surface.
- Keep `node --test` as the release gate.

## Release Criteria

The release is ready when:

- All new commands return structured output on representative fixtures.
- `node --test` passes.
- The regression matrix covers Luau, brain, and workspace command families.
- Release artifacts exist for `0.6.1`.
- The existing 0.6.0 tools still behave unchanged.
