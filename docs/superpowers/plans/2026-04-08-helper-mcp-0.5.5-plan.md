# helper-mcp 0.5.5 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Luau repair, dependency graphs, history/merge tooling, metric snapshots, configurable patterns, and stronger test coverage for `helper-mcp` 0.5.5 without breaking the existing 0.5.0 tool surface.

**Architecture:** Keep the current MCP server and workspace-local storage model. Add a small pattern-loader module so Luau analysis can use `.helper-mcp/patterns.json` overrides, then extend `luau.mjs` for repair, remotes, dependency, complexity, changelog, and metrics reports. Extend `brain.mjs` for history and merge flows, wire the new tools through `src/core.mjs`, and persist new snapshots under `.helper-mcp/` so the server remains stateless outside the workspace.

**Tech Stack:** Node.js ESM, `@modelcontextprotocol/sdk`, filesystem JSON/JSONL persistence, Node test runner, deterministic regex-based analysis.
